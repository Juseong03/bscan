## 2. External-B: circBase 폐기 → sequence-disjoint 감사로 전환 (해결됨)

### 2a. circBase+RJunBase External-B 폐기
circBase 3,000 + RJunBase 3,000으로 시도했으나 **모든 모델 AUC 0.16–0.46** (완벽 반전).

**근본 원인 — 데이터베이스 간 좌표 규약 불일치 (버그 아님):**
| 데이터 | BS canonical | LS canonical |
|--------|:---:|:---:|
| Internal (학습) | 43.3% | 99.7% |
| circBase/RJunBase | 81.0% | 4.0% |

학습 시 모델은 "canonical splice site → LS(0)"을 배웠는데, circBase(BS)는 81% canonical → 모델이 LS로 판정 → 반전. circBase BED와 RJunBase position이 internal과 다른 genomic 위치를 가리킴. `DataSetPrep`을 동일하게 적용해도 입력 좌표 의미가 달라 해결 안 됨. hg19↔hg38 liftOver + DB별 좌표 재앵커링 필요 → **future work로 보류**.

→ `external_data/circbase_b/` 전체 폐기 완료.

### 2b. Sequence-disjoint 감사로 leakage 우려 해소 (대체 완료) ✅
circAtlas(External-A)에서 internal과 서열이 겹치는 샘플을 제거한 subset으로 재평가:

| 항목 | 결과 |
|------|------|
| External-A exact 서열 중복 | **45/8217 (0.5%)** — leakage 거의 없음 |
| Sequence-disjoint subset | 8172/8217 (99.5%) |
| FM 모델 AUC (full → disjoint) | 0.838–0.846 → 0.837–0.845 (**Δ ≈ −0.001**) |
| 전 모델 Δ | −0.0002 ~ −0.0010 (무시 가능) |

**결론: External 일반화 결과는 leakage 때문이 아님을 직접 증명.** reviewer P2 우려 정면 해소.
파일: `research_results/external_b_sequence_disjoint.csv`

## 3. 신규 핵심 발견 (논문 반영 완료)

### 3.1 ALU-density-matched Tier2 — 가장 중요
LS donor를 ALU coverage가 비슷한 것(±5%)으로 매칭해 Tier2 재구성:

| 모델 | Standard Tier2 | ALU-matched | Δ |
|------|:---:|:---:|:---:|
| BSCAN-base | 0.735 ± 0.065 | 0.735 ± 0.061 | +0.001 |
| CircCNN | 0.721 ± 0.041 | 0.724 ± 0.043 | +0.002 |
| BSCAN-FM | 0.478 ± 0.028 | 0.479 ± 0.031 | +0.000 |

**ALU density를 통제해도 Tier2 성능 불변 → ALU는 Tier2 discrimination의 주원인이 아님.**
다른 서열 특징(splice site strength, polypyrimidine tract 등)이 주도.

### 3.2 Multi-scale ALU
| Window | BS has ALU | LS has ALU | inv. pair (BS/LS) | p |
|--------|:---:|:---:|:---:|:---:|
| 100 nt | 8.7% | 7.1% | 0.55% / 0.27% | 9e-7 |
| 250 nt | 23.1% | 19.3% | 3.02% / 1.36% | 3e-14 |
| 500 nt | 40.8% | 36.0% | 9.25% / 5.17% | 7e-18 |

창 크기가 클수록 ALU 신호 강해짐. 500nt에서 inverted pair 1.8× enrichment (ALU-driven 모델 지지).

## 8. Branch Ablation 완료 (2026-06-05) ✅

7개 config 모두 동일 조건(transcript split, 3 seeds, 동일 평가 코드)으로 학습·평가. `results/ablation_summary.csv`.

| Config | Branches | Int AUC | Ext AUC | Drop% |
|--------|----------|:---:|:---:|:---:|
| **Full** | CNN+Stem+Attn | 0.8907 | 0.8496 | 4.6% |
| Full−Attn | CNN+Stem | 0.8838 | 0.8498 | 3.8% |
| Full−Stem | CNN+Attn | 0.8838 | 0.8487 | 4.0% |
| FM+CNN | CNN only | 0.8911 | 0.8450 | 5.2% |
| Full−CNN | Stem+Attn | 0.8695 | 0.7137 | 17.9% |
| FM+Stem | Stem only | 0.8402 | 0.7248 | 13.7% |
| FM+Attn | Attn only | 0.8674 | 0.6852 | 21.0% |

**결론:**
1. **CNN branch가 일반화의 핵심** — 제거 시 ext 0.850→0.714 (최대 손실), CNN 단독으로 거의 풀 성능 재현
2. Stem·Attn은 CNN 있으면 거의 redundant (제거해도 ext 불변), 단독으론 약함
3. Cross-attention 단독이 일반화 최악(21% drop) → exon shortcut에 가장 취약

**주의:** transcript split에서 Full 내부 AUC=0.891 (sample split 0.917보다 낮음 — cross-transcript가 더 어려움). Ablation은 모두 transcript split 3-seed라 상호 공정 비교.

**구현 메모:**
- `bscan_unified_fm_mlponly`(전 branch off)는 degenerate → torch.cat 빈 리스트 에러 → 제외
- trainer.py에 7개 ablation config + fulltr 등록
- evaluate_ablation.py: 내부(transcript) + External-A(circAtlas rnafm 임베딩 재사용) 평가
